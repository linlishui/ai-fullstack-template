# Frontend Review Checklist

本清单用于生成后审查前端质量，不替代 `build` 和 `lint`，而是补足页面质量检查。

## 1. 结构

- 页面是否按 `pages/`、`features/`、`components/`、`api/` 等分层
- `App.tsx` 是否只负责装配
- 是否存在明显的大型单文件组件

## 2. 视觉一致性

- 是否定义了主题 token 或 CSS 变量
- 颜色、圆角、阴影、间距是否统一
- 主按钮、次按钮、危险按钮是否有清晰区分

## 3. 页面质量

- 列表页是否有标题、筛选、结果区、状态区
- 详情页是否有摘要区和关键操作
- 表单页是否有字段分组、字段级错误提示和提交反馈
- 工作台页是否不只是 KPI 数字堆叠

## 4. 状态完整性

- 是否补齐加载态、空态、错误态、禁用态、提交中态、成功反馈
- 接口失败时页面是否有明确提示
- 空数据时是否有下一步引导

## 5. 响应式与可用性

- 移动端是否可操作
- 多列布局是否会在窄屏重排
- 表格或复杂列表在移动端是否有降级方案

## 6. 风险记录

生成项目时，建议在 `generated/<project-slug>/docs/frontend-ui-checklist.md` 中记录：

- 已满足项
- 未满足项
- 视觉风险
- 交互风险
- 待人工确认项
