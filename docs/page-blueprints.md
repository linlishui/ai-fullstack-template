# Page Blueprints

本文件提供页面级蓝图，约束 AI 生成前端时的结构组织方式。

## 1. 列表页

建议包含：

- 页头：标题、说明、主操作按钮
- 筛选区：搜索、筛选、排序、批量操作
- 结果区：表格或卡片列表
- 状态区：加载、空态、错误态

常见错误：

- 只有表格，没有筛选和主操作
- 空态只写“No data”
- 状态标签、时间、负责人等关键信息没有层级

## 2. 详情页

建议包含：

- 头部摘要：名称、状态、关键指标、关键操作
- 主内容区：基础信息、时间线、关联数据
- 辅助区：说明、审计信息、备注、快捷操作

常见错误：

- 详情页只是字段平铺
- 缺少状态变化入口
- 关键操作藏得过深

## 3. 表单页

建议包含：

- 页面说明：当前动作和影响范围
- 表单主体：按业务分组
- 辅助说明：字段解释、约束说明
- 底部操作：取消、保存、提交等

常见错误：

- 所有字段一次性堆满
- 错误提示不贴近字段
- 提交中和成功反馈缺失

## 4. 工作台 / Dashboard

建议包含：

- 顶部摘要指标
- 待处理事项区
- 最近活动或关键提醒
- 快捷操作入口

常见错误：

- 只有统计卡片，没有可操作入口
- 所有模块权重相同，缺乏主次

## 5. 空态和错误态

空态至少应包含：

- 当前为空的原因
- 下一步建议动作
- 返回或创建入口

错误态至少应包含：

- 错误说明
- 重试动作
- 返回路径或联系支持提示
