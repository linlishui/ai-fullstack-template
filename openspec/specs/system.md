# System Base Spec

## 概述

该规格定义当前模板系统自身的基础能力，而不是某个具体业务系统的业务规格。

## 目标

- 提供一个标准化仓库结构
- 提供从需求文档生成 OpenSpec 与代码的提示词体系
- 为后续 AI 自动生成全栈项目提供统一约束

## 基础能力

### 1. 需求输入

- 系统必须以 `requirements/requirement.md` 作为默认需求输入文件
- 需求文档必须支持描述目标、角色、模块、页面、实体、权限、流程、异常和非功能需求

### 2. 规格生成

- 系统必须支持根据需求文档生成 OpenSpec 风格的项目规格
- 规格内容至少包括：需求分析、系统设计、任务拆分、模块边界与数据模型

### 3. 实现生成

- 系统必须支持生成后端、前端、数据库迁移、Redis 集成、测试与容器部署文件
- 系统必须将具体业务实现统一输出到 `generated/<project-slug>/`
- 生成实现时必须遵守环境变量配置、模块化拆分与可验证原则

### 4. 验证与修复

- 系统必须提供生成后的一键检查与修复提示词
- 系统必须提供本地脚本说明如何验证 Docker Compose、测试、Lint 与构建

## 约束

- 当前模板阶段不实现具体业务代码
- 当前模板阶段仅定义流程、文档、脚本和目录约定
- 所有后续业务代码都必须建立在该基础规格之上
